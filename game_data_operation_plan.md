## 一、数据埋点方案：全链路追踪体系

### 1.1 核心商业指标（必须追踪）

| 指标类型 | 具体指标 | 定义 | 目标基准 | 追踪频率 |
|---------|---------|------|---------|---------|
| **留存** | 次日留存 | D1留存率 | ≥45% | 每日 |
| | 7日留存 | D7留存率 | ≥20% | 每日 |
| | 30日留存 | D30留存率 | ≥8% | 每周 |
| | 滚动留存 | D7/D1比值 | ≥45% | 每周 |
| **变现** | ARPDAU | 日活跃用户平均收入 | $0.15-0.25 | 每日 |
| | ARPU | 每用户平均收入 | $2.5-4.0 | 每月 |
| | LTV | 用户生命周期价值 | ≥3×CPI | 动态计算 |
| **获客** | CPI | 单次安装成本 | ≤$1.5(iOS) | 每日 |
| | 自然流量占比 | 非付费安装/总安装 | ≥40% | 每周 |
| | ROI | 投资回报率 | D7≥30%, D30≥100% | 每日 |

**LTV计算公式**：
```
LTV = ARPU × (1 + 留存率D1 + 留存率D7 + 留存率D30 + ...)
目标：LTV / CPI ≥ 3:1（健康盈利线）
```

### 1.2 玩法数据埋点（Roguelike核心）

#### 关卡数据
| 埋点事件 | 触发时机 | 附加参数 |
|---------|---------|---------|
| `level_start` | 开始关卡 | 关卡ID, 难度, 当前卡组 |
| `level_complete` | 通关 | 关卡ID, 用时, 剩余HP, 卡组配置 |
| `level_fail` | 失败 | 关卡ID, 死亡波次/阶段, 死亡原因 |

#### 卡牌数据

### 1.3 付费漏斗追踪

| 漏斗层级 | 关键事件 | 转化目标 | 优化方向 |

| 曝光→点击 | `price_click` | ≥15% CTR | 价格锚点设计 |

| 商店→意向 | `purchase_intent` | ≥40% | 商店UI/UX |
| 意向→支付 | `purchase_complete` | ≥25% | 支付流程简化 |
| 首购→复购 | `purchase_repeat` | ≥35% | 会员/季票设计 |

## 三、付费榜冲刺策略：登顶算法破解

### 3.1 iOS付费榜排名算法解析

**权重估算模型**（基于行业研究）：

```
今日排名 = 
  今日下载量 × 8
+ 昨日下载量 × 5
+ 前日下载量 × 3
+ 大前日下载量 × 2
+ 评分 × 100 × (评论数/1000)
+ 更新奖励 × 50
──────────────────────────────────
              权重总和
```

**关键洞察**：
- 下载量权重是收入的**8-10倍**（付费榜特殊机制）
- 新应用有**7天的新品加权期**
- 更新后48小时内有**排名加权**

### 3.2 首发定价策略

**定价梯度设计**：

| 定价阶段 | 价格 | 时长 | 策略目的 |
|---------|------|------|---------|
| **预约期** | ￥12 / $1.99 | 2周 | 积累预约量，制造稀缺感 |
| **首发价** | ￥18 / $2.99 | 首周 | 锚定价值认知 |
| **恢复价** | ￥30 / $4.99 | 常态 | 长期定价 |
| **促销价** | ￥12 / $1.99 | 限时 | 榜单冲刺/节日 |

**定价心理学技巧**：
1. **9尾数效应**：$2.99 vs $3.00，转化率提升15-20%
2. **价格锚点**：原价￥30划线，现价￥18，感知折扣40%
3. **捆绑销售**：游戏本体+DLC预购包，提升客单价
