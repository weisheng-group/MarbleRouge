| 项目 | 详情 |
|------|------|

| 目标平台 | iOS |

## 1. 🎵 音乐风格定位

### 1.1 整体音乐风格

**核心定位："禅意张力"（Zen Tension）**

我们将采用 **Lo-fi Electronic + 极简主义声学乐器** 的融合风格：

- **基底**：温暖的合成器铺底（类似Celeste的电气氛围，但更柔和）
- **旋律**：稀疏的钢琴或古琴音色，呼应东方"留白"美学
- **节奏**：极慢节奏（60-80 BPM），使用渐入渐出的电子鼓点
- **情绪**：专注但不压抑，放松但有目标感

**参考对比表：**

| 参考游戏 | 借鉴点 | 差异化 |
|---------|--------|--------|
| Celeste | 电子+原声融合，情绪层次感 | 降低紧张度，增加"禅意" |
| Hades | 动态音乐响应机制 | 去掉摇滚的侵略性，保持极简 |
| Stardew Valley | 田园治愈，无压迫感 | 增加策略游戏的思考张力 |
| Monument Valley | 音乐作为叙事语言 | 更强的反馈性和游戏状态响应 |

### 1.2 情绪曲线设计（新手期→中期→后期）

采用 **"波浪渐进"** 而非线性上升，配合Roguelike的"再来一局"循环：

```
情绪曲线图示（单局内）：

准备阶段 → 第1-2回合 → 第3-4回合 → 第5-6回合 → Boss/结算
   🌊        🌊🌊        🌊🌊🌊        🌊🌊🌊🌊       🏆/💀
  平静       蓄力         紧张         高潮        释放
```

**分阶段设计：**

| 阶段 | 音乐层数 | BPM | 核心情绪 | 乐器配置 |
|------|---------|-----|---------|---------|
| **准备/菜单** | 1层 | 60 | 平静、期待 | 单钢琴 + 环境氛围音 |
| **开局（0-25%）** | 2层 | 65 | 专注、探索 | + 极简打击乐 |
| **中期（25-60%）** | 3层 | 70 | 紧张、策略 | + 贝斯线条 + 和声填充 |
| **后期（60-90%）** | 4层 | 75 | 累积、冲刺 | + 合成器arp + 渐入弦乐 |
| **Boss/高潮** | 5层 | 80 | 爆发、决战 | 全奏 + 打击乐加强 |
| **胜利结算** | 2层 | 60 | 满足、轻快 | 钢琴独奏 + 胜利主题变奏 |
| **失败结算** | 1层 | 55 | 遗憾但不沮丧 | 大提琴单音 + 留白 |

**新手→中期→后期的纵向变化：**
- **新手期（0-5小时）**：音乐更"友好"，提示音更明显，降低挫败感
- **中期（5-20小时）**：增加音乐复杂度，解锁新音乐层
- **后期（20小时+）**：引入暗黑变奏版本，满足硬核玩家

### 1.3 循环设计（防疲劳策略）

**核心原则：音乐不能成为"噪音"**

| 策略 | 实现方式 |
|------|---------|
| **微随机化** | 每层音乐有3-4种变奏，每次循环随机播放 |
| **渐入渐出** | 层与层之间3-5秒交叉淡入，避免跳跃感 |
| **智能淡出** | 检测到玩家30秒无操作，音乐自动降至1层（放置友好） |
| **呼吸点** | 每60-90秒插入2-3秒"留白"，重置听觉注意力 |
| **胜利主题** | 胜利时播放完整版主题曲（30秒），建立正向记忆锚点 |

**循环曲目时长建议：**
- 基础层：60-90秒（无缝循环）
- 每层变奏：±5秒随机偏移，制造"活"的感觉
- 胜利主题：30秒（非循环）

## 2. 🔊 音效系统

### 2.1 卡牌打出音效（核心反馈）

**设计原则："一卡一声纹"** ——每种卡牌类型有独特的"声纹"，玩家仅凭声音就能识别卡牌类型。

| 卡牌类型 | 主音色 | 辅助音色 | 情绪 | 示例参考 |
|---------|--------|---------|------|---------|
| **攻击卡** | 木块撞击+金属尾音 | 低频冲击 | 干脆、有力 | 围棋落子 + 刀剑出鞘 |
| **防御卡** | 厚布摩擦 | 低沉共鸣 | 稳重、安全 | 空手道格挡音效 |
| **技能卡** | 水晶共鸣 | 魔法闪光 | 神秘、智慧 | 玻璃杯轻敲泛音 |
| **资源卡** | 硬币落下 | 轻盈叮咚 | 收获、满足 | 金币堆叠声 |
| **诅咒卡** | 纸张撕裂 | 不和谐音 | 负面、警告 | 旧书页撕碎 |
| **Boss卡** | 重低音轰鸣 | 震动反馈 | 压迫、威胁 | 远处雷声 |

**技术细节：**
- 使用 **"音效族"**（Sound Families）：每种类型3-5个变体，随机播放避免重复感
- 增加 **"连击层"**：连续打出同类型卡牌时，音高逐级上升（最多+3 semitone）
- **稀有卡牌**：特殊金光闪烁音效，强化收集满足感

### 2.2 UI交互反馈音

**设计原则："精致但隐形"** ——玩家几乎意识不到这些声音，但去掉会感到"不对劲"。

| 交互类型 | 音量 | 音色 | 时长 | 备注 |
|---------|------|------|------|------|
| 按钮悬停 | 极低（-24dB） | 水滴轻触 | 50ms | 仅触觉反馈，不干扰思考 |
| 按钮点击 | 低（-18dB） | 木块轻敲 | 80ms | 确认感，不抢戏 |
| 切换标签 | 中（-12dB） | 纸张滑动 | 150ms | 有明显的"切换"感知 |
| 拖拽卡牌 | 动态 | 轻微摩擦 | 持续 | 随拖拽速度变化音高 |
| 卡牌放置 | 中（-12dB） | 对应卡牌类型 | 100ms | 与卡牌音效呼应 |
| 获得奖励 | 高（-6dB） | 金币+铃铛 | 500ms | 正向强化，可稍明显 |
| 错误提示 | 中（-12dB） | 闷音木块 | 150ms | 不刺耳，但明确拒绝 |

**防烦躁设计：**
- 同一音效连续触发间隔 >100ms（防连点刷屏）
- 设置 **"音效密度控制"**：1分钟内UI音效超过20个自动降低50%音量
- 提供 **"极简模式"** 选项：仅保留卡牌和胜负音效

### 2.3 胜利/失败/升级奖励音效

**胜利音效（核心钩子）：**

```
胜利音效结构（3秒）：
├── 触发：最终伤害数字定格
├── 0.0s - 时间冻结效果音（200ms）
├── 0.2s - 胜利和弦奏响（C大调，明亮）
├── 0.5s - 金币雨落下（持续1.5s）
├── 1.0s - 升级提示音（如有升级）
└── 2.5s - 胜利主题片段淡出，引导结算
```

- 胜利主题采用 **"耳虫旋律"**（Earworm）——3-4个音符的极简旋律，容易记住
- 不同难度胜利有不同变奏：普通胜利→铜色混响；困难胜利→金色混响；完美胜利→钻石音色

**失败音效（降低挫败感）：**

```
失败音效结构（2.5秒）：
├── 触发：生命值归零
├── 0.0s - 低音下滑（表示"下沉"）
├── 0.5s - 大提琴单音（遗憾但不悲伤）
├── 1.5s - "再来一局"暗示音（轻微上升，给予希望）
└── 2.0s - 完全静默，等待玩家操作
```

- **关键**：失败音效后必须有1-2秒静默，给玩家"接受现实"的心理缓冲
- 失败音效 **绝对避免** 刺耳、嘲弄的音色

**升级奖励音效：**

| 奖励类型 | 音效设计 |
|---------|---------|
| 解锁新卡牌 | 打开宝盒 + 神秘闪光 |
| 角色升级 | 力量上升音阶（类似RPG升级） |
| 获得货币 | 金币堆叠（数量越多声音越厚重） |
| 成就解锁 | 史诗感和弦 + 短暂静默 + 提示音 |
| 新皮肤/装饰 | 魔法变换音（惊喜感） |

## 3. 🎧 空间音频 / 3D音效

### 3.1 是否需要？

**结论：不需要完整3D空间音频，但需要2.5D层次设计**

| 因素 | 分析 |
|------|------|
| 游戏类型 | 2D卡牌游戏，视觉焦点在屏幕中心 |
| 玩家习惯 | iOS玩家多数使用耳机或手机扬声器，立体声足够 |
| 开发成本 | 3D音频开发成本×2，收益不成正比 |
| 放置元素 | 背景音乐需要"包围感"而非定位感 |

### 3.2 2.5D音频层次设计

采用 **"伪3D分层"** 替代真3D：

```
音频层次（左右声道分布）：

左声道 ──────────────────────────────────── 右声道
    │              中央区                │
    │    背景音乐（70%）                  │
    │    核心音效（卡牌、胜负）            │
    │                                    │
    │    侧翼区                          │
    │    次要UI音效（15%偏置）            │
    │    环境氛围（随机漂移）              │
```

**具体实现：**
- **中央区（Center）**：背景音乐、卡牌音效、胜负音效——单声道或轻微立体声
- **侧翼区（Side）**：次要UI音效、通知音效——轻微左右偏移（±20%）增加空间感
- **动态平移**：Boss出场时，音效从左到右扫过，制造"压迫逼近"感

### 3.3 对体验的提升

| 体验维度 | 2.5D音频的作用 | 预期提升 |
|---------|---------------|---------|
| **沉浸感** | 环境音轻微漂移，模拟"活"的世界 | +15% |
| **信息层级** | 重要声音中央聚焦，次要声音散开 | +25% |
| **疲劳度** | 立体声分布减轻单声道压迫感 | +20% |
| **Boss战** | 特殊音效平移制造紧张感 | +30% |

## 4. ⚙️ 技术规格

### 4.1 音频引擎选择

| 引擎 | 评分 | 适用性分析 |
|------|-----|-----------|
| **Unity Audio** | ⭐⭐⭐⭐☆ | 推荐。足够满足需求，包体最小，iOS原生支持最好 |
| **FMOD** | ⭐⭐⭐☆☆ | 功能强大但过度设计，增加2-3MB包体 |
| **Wwise** | ⭐⭐☆☆☆ | 3A级工具，小型项目性价比低 |

**推荐方案：Unity Audio + 自定义分层系统**

选择理由：
1. **包体大小**：Unity Audio无需额外库，节省1.5-3MB（iOS用户敏感）
2. **性能**：卡牌游戏音频需求简单，原生音频足够
3. **迭代速度**：无需学习第三方工具，音频师可直接在Unity中工作
4. **维护**：减少外部依赖，降低长期维护成本

### 4.2 包体大小控制（iOS优化）

**目标：音频总大小 < 15MB**

| 类别 | 原始大小 | 压缩后目标 | 压缩策略 |
|------|---------|-----------|---------|
| 背景音乐（5首×3层） | 30MB | 6MB | AAC 128kbps，循环点优化 |
| 音效库（150个） | 15MB | 3MB | Vorbis 96kbps，短音效 |
| UI音效（50个） | 5MB | 1MB | ADPCM，更低延迟 |
| 语音/旁白（如有） | 10MB | 2MB | AAC 96kbps |
| **总计** | **60MB** | **12MB** | - |

**具体策略：**
- **音乐**：使用 **无缝循环压缩**，将长音乐切分为可循环的小段
- **音效**：建立 **音效复用池**，相似音效共用底层采样
- **流式加载**：Boss音乐等大文件采用流式，不常驻内存
- **质量分级**：高清版本商店截图展示，实际下载使用压缩版

### 4.3 动态混音系统（Ducking & Layering）

**核心架构：**

```
混音架构（Audio Mixer Groups）：

Master
├── BGM（背景音乐）- 优先级：低
│   ├── BGM_Base（基础层）
│   ├── BGM_Mid（中层）
│   ├── BGM_High（高层）
│   └── BGM_Boss（Boss专用）
├── SFX_Game（游戏音效）- 优先级：高
│   ├── SFX_Cards（卡牌）
│   ├── SFX_Combat（战斗）
│   └── SFX_Rewards（奖励）
├── SFX_UI（UI音效）- 优先级：中
│   └── （内部分类）
└── Ambient（环境音）- 优先级：最低
```

**动态规则：**

| 场景 | BGM音量 | SFX_Game | SFX_UI | 说明 |
|------|--------|----------|--------|------|
| 正常游戏 | 0dB | 0dB | -6dB | 标准状态 |
| 卡牌连击 | -3dB | +2dB | -12dB | 突出游戏反馈 |
| Boss战 | 0dB | +3dB | -18dB | 全焦点战斗 |
| 放置模式（30秒无操作） | -6dB | -12dB | -6dB | 音乐为主导 |
| 结算画面 | -12dB | 0dB | 0dB | 突出奖励反馈 |

**Snapshot系统：**
- 预设5种混音快照（Normal/Combat/Boss/Reward/Idle）
- 快照切换过渡时间：0.5-1秒（平滑渐变）

## 6. 🎯 核心体验强化："再来一局"钩子

### 6.1 音频如何强化核心循环

**"再来一局"的心理学链条：**

```
胜利 → 奖励音效+主题音乐 → 多巴胺释放 → 情绪高点
   ↓
结算画面音乐（期待感）
   ↓
"再来一局"按钮音效（轻微上升音）→ 行为触发
   ↓
下一句准备音乐（熟悉感+新鲜感）→ 循环开始
```

**具体设计：**

| 钩子点 | 音频设计 | 心理机制 |
|-------|---------|---------|
| **胜利主题** | 3-4音符极简旋律，易记忆 | 建立"胜利=这首歌"的条件反射 |
| **奖励音效** | 金币+升级的组合音效 | 强化正向行为反馈 |
| **结算等待** | 音乐渐弱但不停，保持注意力 | 降低退出意愿 |
| **再来一局按钮** | 特殊音效（与其他按钮不同） | 听觉引导，降低决策成本 |
| **开局音乐** | 每次开局略有不同（微随机） | 新鲜感+熟悉感的平衡 |

### 6.2 A/B测试建议

| 测试项 | 变体A | 变体B | 评估指标 |
|-------|------|------|---------|
| 胜利音效 | 完整主题（3秒） | 极简提示（1秒） | 再来一局点击率 |
| 失败音效 | 带希望暗示 | 完全静默 | 退出率vs重试率 |
| 背景音乐 | 4层渐进 | 2层简化 | 平均游戏时长 |
| 音效密度 | 标准 | +30% | 疲劳度评分 |
