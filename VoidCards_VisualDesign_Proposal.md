# Void Cards 视觉设计提案 + UI/UX规范文档

## 🎮 项目定位
- **游戏类型**: Roguelike卡牌 + 极简放置
- **目标平台**: iOS (主打) / Android
- **目标用户**: 25-35岁，喜欢策略游戏，追求美学体验
- **商业目标**: iOS付费榜第一

---

## 一、美术风格定位

### 1.1 核心风格选择：「禅意几何·极简东方」

我们选择了**Monument Valley式的极简几何** + **东方禅意美学**的融合风格。

**为什么是这种风格？**
- ✅ 市场差异化：卡牌游戏市场充斥着暗黑手绘(Slay the Spire)和像素风(Dead Cells)
- ✅ 移动端优势：简洁几何在高分辨率屏幕上表现力极强
- ✅ 受众契合：目标用户偏好安静、有深度的美学体验
- ✅ 开发友好：几何风格资产生产效率高，适合小团队

### 1.2 视觉关键词

**三个词概括：空灵 · 几何 · 呼吸感**

| 关键词 | 解释 | 视觉体现 |
|--------|------|----------|
| 空灵 | 留白、静谧、冥想感 | 大面积负空间、极简配色 |
| 几何 | 数学美感、秩序感 | 圆角三角形、六边形、黄金比例分割 |
| 呼吸感 | 动态节奏、生命力 | 微妙的浮动动画、渐变呼吸效果 |

### 1.3 配色方案

#### 主色调（禅意色系）
```
主色 - 雾靛蓝:     #3D5A80 (稳重、可信赖)
主色 - 石青:       #98C1D9 (清澈、透明感)
```

#### 辅助色（自然色系）
```
辅助色 - 雾白:     #E0FBFC (纯净、空间感)
辅助色 - 石墨灰:   #293241 (深度、神秘感)
辅助色 - 砂岩:     #EE6C4D ( warmth，极少使用)
```

#### 强调色（情绪标记）
```
攻击/危险:    #E07A5F (珊瑚红)
防御/护盾:    #81B29A (薄荷绿)
能量/法力:    #F2CC8F (琥珀金)
特殊/传说:    #B8A9C9 (薰衣草紫)
```

#### 色彩使用原则
1. **60-30-10法则**：60%雾白/石墨灰 + 30%雾靛蓝/石青 + 10%强调色
2. **动态配色**：根据游戏阶段微调色相（战斗时偏冷色，胜利时偏暖色）
3. **高对比度**：确保色盲用户可区分（WCAG AA标准）

## 二、UI系统架构

### 2.1 设计理念：「隐形UI」

> "最好的UI是玩家感受不到它的存在" —— 让游戏世界本身成为界面

- **沉浸优先**：UI元素尽量融入场景
- **即时反馈**：每个交互都有视觉/听觉/触觉反馈
- **减少认知负担**：一眼看懂，无需思考

### 2.2 主界面布局

```
┌─────────────────────────────────────┐
│  ┌─────┐                           │
│  │头像 │  玩家名称                  │ ← 顶部状态栏 (10%)
│  └─────┘  ⚡能量  ❤️生命  💰金币   │
├─────────────────────────────────────┤
│                                     │
│         [  游戏Logo/艺术字  ]        │
│                                     │
│    ┌─────────────────────────┐      │
│    │    当前进行中的游戏     │      │ ← 中央内容区 (70%)
│    │    [  继续冒险  ]      │      │
│    └─────────────────────────┘      │
│                                     │
│    ┌──────┐  ┌──────┐  ┌──────┐   │
│    │ 每日 │  │ 图鉴 │  │ 设置 │   │ ← 快捷入口
│    │ 挑战 │  │ 收集 │  │ ⚙️  │   │
│    └──────┘  └──────┘  └──────┘   │
│                                     │
├─────────────────────────────────────┤
│  [🏆成就]  [📊统计]  [🛒商店]      │ ← 底部导航栏 (20%)
└─────────────────────────────────────┘
```

#### 信息层级
1. **L1 - 核心信息**：生命值、能量（永远可见，小但清晰）
2. **L2 - 导航信息**：主按钮、快捷入口（中等大小，清晰分隔）
3. **L3 - 辅助信息**：成就、设置、统计（可折叠或次级页面）

### 2.3 战斗界面设计

```
┌─────────────────────────────────────┐
│  🏆 第12层    ⚡3/3    ❤️45/50     │ ← 战斗状态栏
├─────────────────────────────────────┤
│                                     │
│         ┌─────────────┐             │
│         │   敌人A     │             │ ← 敌人区域 (上方)
│         │   [动画]    │             │
│         │   ❤️30     │             │
│         └─────────────┘             │
│                                     │
│    场地效果: [护盾] [燃烧] [冰冻]    │ ← 战场状态 (中部)
│                                     │
├─────────────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐│
│  │卡1 │ │卡2 │ │卡3 │ │卡4 │ │卡5 ││ ← 手牌区域 (下方)
│  │⚡1 │ │⚡2 │ │⚡0 │ │⚡1 │ │⚡3 ││   弧形排列，可滑动
│  └────┘ └────┘ └────┘ └────┘ └────┘│
├─────────────────────────────────────┤
│  [结束回合]    [设置]  [逃跑?]      │ ← 操作栏
└─────────────────────────────────────┘
```

#### 卡牌展示规范
- **尺寸**：宽120pt × 高180pt（iPhone标准）
- **圆角**：16pt（符合iOS Human Interface Guidelines）
- **卡牌层级**：
  - 顶层：名称 + 能量消耗（左上角圆形标记）
  - 中层：抽象符号插画（几何构成）
  - 底层：类型底色 + 描述文字
- **手势交互**：
  - 长按预览放大（1.5倍）
  - 上滑打出
  - 左右滑动切换选择

#### 交互反馈设计
| 操作 | 视觉反馈 | 触觉反馈 | 音效 |
|------|----------|----------|------|
| 选中卡牌 | 卡牌放大1.1倍 + 发光边框 | Light Impact | 纸张轻拂 |
| 拖动卡牌 | 卡牌倾斜 + 粒子拖尾 | 无 | 空气流动 |
| 打出卡牌 | 飞入战场 + 消散粒子 | Medium Impact | 元素释放 |
| 受到伤害 | 屏幕红边脉冲 + 震动 | Heavy Impact | 低沉轰鸣 |
| 获得能量 | 能量球飞入UI | Light Impact | 清脆叮铃 |

#### 结构层级
```
主菜单
├── 开始游戏
│   ├── 选择角色 → 开始冒险
│   └── 继续游戏
├── 每日挑战
│   └── 排行榜
├── 卡牌图鉴
│   ├── 按职业筛选
│   └── 按类型筛选
├── 成就系统
│   ├── 进度总览
│   └── 已完成
├── 统计数据
│   ├── 总览
│   ├── 职业统计
│   └── 最近战斗
└── 设置

    ├── 音乐
    ├── 通知
    ├── 语言
    ├── 账号

```

#### 导航动画规范
- **进入子菜单**：当前页面向左滑出（-30%透明度），新页面从右侧滑入
- **返回上级**：当前页面向右滑出，上级页面从左侧滑入恢复
- **转场时长**：300ms
- **缓动曲线**：ease-out-cubic

## 三、动效设计

### 3.1 动效设计原则

1. **有意义**：每个动画都有目的（引导注意力、确认操作、展示状态）
2. **克制**：动画时长控制在200-500ms之间
3. **流畅**：保持60fps，支持Reduce Motion无障碍设置
4. **一致性**：相同类型的交互使用相同的动画模式

### 3.2 卡牌打出反馈

#### 阶段1：选中（0-100ms）
- 卡牌放大至110%
- 添加金色发光边框（强调色）
- 其他卡牌透明度降至70%
- 可能的攻击目标高亮显示

#### 阶段2：拖动（100-300ms）
- 卡牌跟随手指/光标
- 添加运动模糊（微妙）
- 释放目标时触发目标确认动画

#### 阶段3：释放（300-500ms）
- 卡牌飞向目标（抛物线轨迹）
- 飞行中卡牌逐渐变小并旋转
- 撞击时产生粒子爆发

#### 阶段4：结算（500-800ms）
- 效果数字弹出（伤害/治疗/护盾）
- 数字向上浮动并逐渐消失
- Combo连击时数字样式升级（普通→银色→金色→彩虹）

### 3.3 胜利/失败情绪化动效

#### 胜利动画「升华为光」
```
时长: 3秒
序列:
  1. 敌人消散为粒子 (0-500ms)
  2. 粒子重组为光球上升 (500-1500ms)
  3. 光球爆炸为满天星斗 (1500-2500ms)
  4. "胜利"文字从星光中浮现 (2500-3000ms)
  
特效:
  - 屏幕整体色调转为暖色
  - 震动：成功音效的重拍点
  - 解锁内容卡片依次飞入
```

#### 失败动画「归于虚无」
```
时长: 2秒
序列:
  1. 屏幕逐渐变为黑白 (0-800ms)
  2. 画面从边缘开始"侵蚀"为像素 (800-1500ms)
  3. 最后画面碎裂消散 (1500-2000ms)
  
特效:
  - 低沉的嗡鸣声
  - 轻微震动
  - "再来一次"按钮从下方升起
```

### 3.4 转场动画风格

#### 场景切换「门户穿越」
- 当前场景缩小并进入中心的几何传送门
- 传送门旋转并切换颜色
- 新场景从传送门中"流出"并放大填充屏幕

#### 战斗开始「聚焦」
- 地图视图模糊并缩小
- 战斗场景从中心清晰浮现
- 敌人和玩家单位依次入场（ staggered 200ms间隔）

#### 菜单切换「层叠滑动」
- 使用iOS标准的Navigation Controller动画
- 添加微妙的内容模糊效果（Backdrop Blur）
- 导航栏元素平滑过渡

### 3.5 放置元素动效

虽然是Roguelike，但加入放置元素后：

#### 离线收益展示
```
回到游戏时:
  1. 暗色遮罩覆盖全屏
  2. "你离开期间"文字淡入
  3. 资源图标依次弹跳出现，数值滚动增长
  4. "点击继续"按钮发光提示
```

#### 自动战斗模式
- 切换时UI简化（隐藏手牌，仅保留核心信息）
- 卡牌自动打出时有"幽灵手"的提示动画
- 可随时点击"接管"立即切换回手动

## 四、视觉识别系统

### 4.1 App Icon设计方案

#### 方案A：「虚空之瞳」⭐推荐
- **设计**：同心圆构成的抽象眼睛，中心是发光的几何核心
- **颜色**：深灰背景 (#293241) + 雾靛蓝渐变 + 中心金色光点
- **特点**：在App Store中极具辨识度，传达"洞察"和"策略"感
- **适应性**：缩小时仍清晰，支持暗黑模式变体

#### 方案B：「悬浮卡牌」
- **设计**：一张发光的卡牌悬浮在几何底座上
- **颜色**：浅色背景 + 卡牌雾靛蓝
- **特点**：直接传达游戏类型，但可能与其他卡牌游戏混淆

#### 方案C：「螺旋上升」
- **设计**：六边形螺旋向上的抽象塔
- **颜色**：蓝紫渐变
- **特点**：暗示Roguelike的"爬塔"核心

### 4.2 App Store截图策略

#### 前5张截图故事线

**图1 - 主视觉封面**
- 内容：游戏Logo + 核心角色 + 标志性场景
- 文案：「一场极简的策略之旅」
- 目标：建立第一印象，传达美学风格

**图2 - 核心玩法展示**
- 内容：战斗界面，展示卡牌打出瞬间
- 文案：「每一张牌，都改变战局」
- 目标：3秒内让玩家理解游戏类型

**图3 - 策略深度**
- 内容：卡组构建界面，展示多张卡牌组合
- 文案：「Build Your Deck, Forge Your Fate」
- 目标：吸引核心卡牌游戏玩家

**图4 - 放置玩法**
- 内容：离线收益界面，展示成长感
- 文案：「即使离线，战斗仍在继续」
- 目标：吸引放置游戏爱好者

**图5 - 社交/成就**
- 内容：排行榜或每日挑战界面
- 文案：「挑战全球玩家，攀登高峰」
- 目标：激发竞争欲和下载动力

#### 截图设计规范
- **尺寸**：iPhone 6.7" Display (1290×2796)
- **风格**：使用真实游戏画面，顶部叠加文案
- **字体**：SF Pro Display Bold（与iOS原生一致）
- **本地化**：准备10+语言版本

### 4.3 辅助图形系统

#### Logo设计
- **主Logo**：文字"Void Cards"使用定制几何无衬线字体
- **简化版**：仅使用"VC"字母组合，适用于小尺寸
- **动态Logo**：启动画面时Logo有呼吸动画

#### 图标系统
使用统一的图标设计语言：
- 所有图标基于 24×24 网格
- 2pt 描边，圆角端点
- 风格：线性与面性混合，根据场景切换

#### 插画风格
- 角色：抽象几何化表达，无具体五官
- 场景：等角透视的几何建筑
- 特效：粒子 + 光线追踪效果

## 五、技术实现建议

### 5.1 美术管线

#### 工具链推荐
| 环节 | 推荐工具 | 备选方案 | 预算 |
|------|----------|----------|------|
| 矢量设计 | Figma | Adobe Illustrator | 免费/$20月 |
| 动效设计 | After Effects | Rive | $20月/免费 |
| 原型制作 | ProtoPie | Principle | $11月/$99年 |
| 游戏内实现 | Unity UI Toolkit | UIWidgets | 已包含 |

#### 资源生产流程
```
概念设计 (Figma) → 动效原型 (After Effects) → 
切图导出 (1x/2x/3x) → Unity实现 → 
真机测试 → 迭代优化
```

### 5.2 性能优化规范

#### 图形性能
- 所有UI元素使用图集(Atlas)打包
- 限制最大图集尺寸 2048×2048
- 动态元素使用对象池
- 粒子效果限制最大粒子数 50

#### 内存管理
- 图片格式：iOS使用ASTC压缩
- 字体：仅打包使用的字符子集
- 音效：压缩为MP3/OGG，背景音乐流式加载

### 5.3 外包建议

#### 必须内部完成
- UI框架搭建
- 核心交互实现
- 动效系统集成

#### 可外包内容
| 内容 | 预算范围 | 交付标准 |
|------|----------|----------|
| 角色概念设计 | $2000-5000 | 3个主角+10个敌人 |
| 场景背景 | $1500-3000 | 5个主题场景 |
| 音效设计 | $1000-2000 | 完整SFX库 |
| BGM音乐 | $2000-4000 | 5首循环曲目 |

#### 外包平台推荐

### 5.4 无障碍设计

#### 视觉无障碍
- [ ] 支持动态字体大小（iOS动态类型）
- [ ] 所有交互元素支持VoiceOver
- [ ] 色盲友好模式（高对比度模式）
- [ ] 支持Reduce Motion减少动画

#### 触觉反馈
- 为关键操作添加Haptic Feedback
- 支持不同程度的震动（Light/Medium/Heavy）
